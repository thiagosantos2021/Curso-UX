User Researcher

Vertical de pesquisa da área de UX, o User Researcher é o responsável por fazer as perguntas certas para identificar as motivações, necessidades e dores dos usuários. Depois, precisam analisar os resultados com o objetivo de identificar os comportamentos, criar personas e jornadas do usuário.

Com base nas descobertas, eles ajudam a guiar o desenvolvimento do produto digital. Isso significa trabalhar junto dos Product Managers (gerentes de produto) e desenvolvedores. Seja na etapa de pesquisa ou de compartilhamento dos resultados com as equipes, o UX Researcher lida com gente. Ou seja, quem escolhe essa carreira precisa gostar da ideia de passar bastante tempo com pessoas.

Tipos de Pesquisa e Metodologias

Há diversos métodos de pesquisa UX. Dito isso, alguns deles podem ser mais apropriados que outros para o seu projeto, dependendo das suas restrições de tempo, ferramentas de usa e tipo de produto ou serviço. Uma dica é usar métodos diferentes ou alternados a cada ciclo do produto, pois eles visam diferentes objetivos e tipos de insight em cada etapa. Abaixo, listamos quais os principais métodos de pesquisa UX em cada etapa (e ah, não é uma regra, afinal, você pode criar etapas que fazem sentido para seu projeto).

Descobrir: este estágio é quando você tenta esclarecer o que não sabe e entender melhor o que as pessoas precisam. Os métodos dele são, por exemplo:

  • Estudo de campo;
  • Entrevista com o(s) usuário(s);
  • Entrevista com Stakeholders;
  • Coleta de requisitos e restrições do projeto.

Explorar: serve para entender o espaço do problema e o escopo do projeto e atender às necessidades do usuário adequadamente.Os métodos dele são, por exemplo:

  • Análise competitiva;
  • Revisão do Design;
  • Construção de personas;
  • Análise de tarefas;
  • Feedback e teste de protótipo (protótipos clicáveis ​​ou em papel);
  • Mapeamento de jornada.

Teste: são para verificar projetos durante o desenvolvimento e para garantir que os sistemas funcionem bem para as pessoas que os usam, neste, os usuários/clientes do seu serviço/produto. Os métodos dele são, por exemplo:

  • Avaliação de acessibilidade;
  • Teste de benchmark;
  • Teste qualitativo de usabilidade.

Ouvir: se aplicado durante todo o processo de desenvolvimento do seu produto ou serviço, vai ajudá-lo a entender os problemas existentes e procurar novos problemas. Desta forma, você poderá analisar os dados coletados e monitorá-los em busca de padrões e tendências. Os métodos dele são, por exemplo:

  • Pesquisa;
  • Análise analítica;
  • Análise de erros de usabilidade;
  • Revisão de perguntas frequentes (FAQ).

Problem Statement e Questionário

Um Problem Statement (ou declaração de problema) identifica a lacuna entre o estado atual (ou seja, o problema) e o estado desejado (ou seja, o objetivo) de um processo ou produto. Dentro do contexto do design, você pode pensar no problema do usuário como uma necessidade não atendida. Ao projetar uma solução que atenda a essa necessidade, você pode satisfazer o usuário e garantir uma experiência agradável ao usuário.

Já um questionário é um método de pesquisa assim como entrevistas, observações, experimentação, grupos focais, etc. Não há um método universal, cada um deles terá um objetivo e cabe a você definir qual será o mais adequado ao seu projeto. Com isso, antes de defini-lo, pense no tempo que levará, nos dados que você pode obter e na riqueza de informações que espera receber.

Como recrutar respondentes

Mesmo com seu projeto de UX design, se você não sabe se seus resultados são utilizáveis e práticos, não poderá garantir uma boa experiência de usuário, não é mesmo? Por isso, a pesquisa é importante, mas como recrutar usuários?

Aqui, não há regras, mas para cada tipo de estudo de usabilidade que envolve usuários, você precisará determinar o método que utilizará. Para isso, há cinco métodos principais de teste de usabilidade:

Testes de guerrilha: como o nome sugere, aqui você irá à “guerra” atrás de usuários. Este é o método mais simples. Nele, basicamente, você pergunta às pessoas se gostariam de participar de sua pesquisa. Podem ser seus amigos, familiares ou colegas de trabalho.

Usuários existentes: este método é aplicado quando você quer fazer o teste que já tenha usuários ativos, por exemplo, como um site. Neste cenário, você poderá a base de usuários existentes para desenvolver os testes, utilizando técnicas de pop-up ou jornadas de email.

Serviço de recrutamento online: com este método você poderá contar com auxiliar de agências ou serviços online para encontrar usuários para participar de sua pesquisa. Plataformas como, Craigslist, Usertesting.com, Zero Bug e Lumens Consultoria disponibilizam esse serviço.

Banco de dados de usuários: este método consiste em utilizar empresas que possuem uma grande quantidade de dados de usuários em seu sistema (como dados demográficos) para segmentar sua pesquisa.

Empresas de recrutamento: por fim, empresas de recrutamento. Essa é, de longe, a maneira mais cara de conseguir usuários para pesquisa, porém, é uma das mais eficientes. O recrutador será responsável por auxiliá-lo elaborar um bom perfil de recrutamento assim como encontrar pessoas para participar.
